# 데이터 분석 결과 (방향 반영)

## 핵심 결론
**현재 데이터로는 1.5m @ 90% 달성 불가능**

## 방향 반영 전 vs 후 비교

### 방향 무시 (위치만)
```
총 그리드: 491개
평균 샘플: 22.1개
중앙값: 13개
샘플 < 5개: 102개 (20.8%)
```

### 방향 반영 (위치 + 방향)
```
총 조합: 1,379개
평균 샘플: 7.9개
중앙값: 4개
샘플 < 5개: 780개 (56.6%)
```

## 샘플 분포 (방향 반영)

| 샘플 수 | 조합 개수 | 비율 |
|---------|-----------|------|
| 0 ~ 5개 | 780개 | 56.6% |
| 5 ~ 10개 | 269개 | 19.5% |
| 10 ~ 15개 | 135개 | 9.8% |
| 15 ~ 20개 | 69개 | 5.0% |
| 20 ~ 30개 | 63개 | 4.6% |
| 30 ~ 50개 | 40개 | 2.9% |
| 50 ~ 100개 | 22개 | 1.6% |
| 100+개 | 1개 | 0.1% |

## 문제점

### 1. 샘플 부족 (56.6%)
- **780개 조합이 5개 미만** = 전혀 학습 안 됨
- 76.1% 조합이 10개 미만 = 매우 약하게 학습

### 2. 방향별 불균형
- 같은 위치라도 방향마다 샘플 수 천차만별
- 예: X=-26m, Y=0.45m
  - 서쪽 방향: 3개
  - 남서쪽 방향: 3개
  - 북쪽 방향: 3개
  - 동쪽 방향: 2개
  → 모든 방향이 샘플 부족

### 3. 극심한 불균형
- 표준편차 11.1 (평균 7.9)
- 최소 1개 ~ 최대 100개
- 중앙값 4개 = 절반은 4개 이하

## 예상 성능

### 현실적 예상
```
샘플 많은 조합 (20+개): 1.5~2.0m 가능
샘플 중간 조합 (10~20개): 2.0~3.0m
샘플 적은 조합 (5~10개): 3.0~5.0m
샘플 거의 없는 조합 (<5개): 5.0m+
```

### 전체 평균 (방향 고려)
```
56.6% × 5m + 19.5% × 4m + 24% × 2m
≈ 3.5~4.0m @ 90%
```

**1.5m @ 90%는 불가능**

## 필요한 조치

### Option 1: 데이터 추가 수집
- 샘플 < 10개 조합 (76.1% = 1,049개)
- 각 조합마다 최소 20개 이상 필요
- **필요 추가 샘플: 약 15,000개**

### Option 2: 방향 무시하고 학습
- 방향을 입력 feature로 추가 (Yaw만 사용)
- 모델이 방향과 위치 관계를 학습하도록
- 예상 성능: 2.5~3.0m @ 90%

### Option 3: 목표 하향 조정
- 현실적 목표: **2.5~3.0m @ 90%**
- 또는 샘플 많은 구간만 평가: **1.5~2.0m @ 90%**

## 권장 사항

**즉시 실행:**
1. 방향 무시하고 학습 (Yaw를 feature로 포함)
2. 성능 측정

**장기적:**
