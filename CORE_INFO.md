# 핵심 정보 - 절대 까먹지 말 것

## 목표
- **1.5m @ 90%tile**: 90% 샘플이 1.5m 이내 오차
- 최종적으로 1.5m 정확도 달성 필요

## 데이터 구조
- 입력: (250, 6) - MagX, MagY, MagZ, Pitch, Roll, Yaw
- 출력: (x, y) - 실제 위치 좌표
- Window: 250 샘플 (5초 @ 50Hz)
- Stride: 50 샘플 (1초)

## ⚠️ 데이터 분석 결과 (중요!)
**방향 반영시:**
- 총 1,379개 (그리드, 방향) 조합
- 평균 샘플: 7.9개 (매우 적음)
- 중앙값: 4개 (절반이 4개 이하!)
- **56.6%가 샘플 < 5개** = 학습 불가능
- **현재 데이터로 1.5m @ 90% 달성 불가능**
- **예상 성능: 3.5~4.0m @ 90%**

**방향 무시시:**
- 총 491개 그리드
- 평균 샘플: 22.1개
- 40.5%가 샘플 < 10개
- **예상 성능: 2.5~3.0m @ 90%**

## 핵심 문제
1. **방향성 문제**: 1→11과 11→1은 같은 위치(x,y)지만 다른 지자기 값
   → 방향 구분하면 데이터 부족 심각
2. **데이터 불균형**: 56.6% 조합이 샘플 < 5개
3. **현실적 접근**: 방향 무시하고 Yaw를 feature로 포함
